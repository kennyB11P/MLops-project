# Анализ метрик экспериментов классификации отзывов

Этот README объединяет результаты двух этапов экспериментов:

1. **Embedding-модели + классические классификаторы** — основной эксперимент с готовыми sentence embeddings и моделями `LinearSVC`, `Ridge`, `SGD`, `LogReg`, `RandomForest`, `ExtraTrees`.
2. **RuBERT frozen encoder** — нейросетевой эксперимент: замороженный `DeepPavlov/rubert-base-cased` + обучаемая линейная голова.

Главная задача обоих этапов — multi-label классификация отзывов по проблемным классам. Один отзыв может иметь несколько labels, поэтому сумма `support` по классам больше числа отзывов в golden set.

## 3. Как читать метрики

В этой задаче особенно важны три группы метрик:

- **macro_precision / macro_recall / macro_f1** — среднее качество по классам. Это главная группа метрик, потому что классы несбалансированы.
- **micro_precision / micro_recall / micro_f1** — качество по всем предсказаниям сразу. Сильнее зависит от частых классов.
- **weighted_precision / weighted_recall / weighted_f1** — среднее по классам с весом по `support`, то есть частые классы влияют сильнее.

При выборе модели основной акцент сделан на **macro_f1**, поскольку классы несбалансированы и важно учитывать качество не только на частых, но и на редких категориях.

# Этап 2. RuBERT frozen encoder

## 10. Что проверялось

В этом эксперименте использовалась модель:

```text
DeepPavlov/rubert-base-cased
```

Схема:

```text
текст отзыва
    ↓
замороженный RuBERT encoder
    ↓
CLS / pooled features
    ↓
обучаемая линейная multi-label голова
```

Encoder не дообучался, обучалась только классификационная голова. Поэтому это не полноценный fine-tuning RuBERT, а более легкая схема обучения.

## 11. Общие метрики RuBERT

| experiment                             |   macro_precision |   macro_recall |   macro_f1 |   micro_f1 |   weighted_f1 |

| RuBERT_frozen_head__threshold_0_5      |             0.459 |          0.756 |      0.562 |      0.57  |         0.569 |

### Анализ RuBERT summary

Вариант с `threshold = 0.5` дал:

```text
macro_f1 = 0.562
```

Вариант с подобранными thresholds на validation set дал:

```text
macro_f1 = 0.564
```

Улучшение почти нулевое. Threshold tuning повысил precision, но снизил recall:

```text
macro_precision: 0.459 → 0.510
macro_recall:    0.756 → 0.701
```

То есть модель стала осторожнее, но начала пропускать больше релевантных labels. Weighted F1 при этом ухудшился:

```text
weighted_f1: 0.569 → 0.556
```

Поэтому для RuBERT threshold tuning нельзя назвать уверенно полезным. Он немного повышает macro F1, но ухудшает качество по частым классам.

## 21. Итог по RuBERT head-only classifier

Конфигурация `замороженный RuBERT + линейная голова` на малом и несбалансированном train set не обеспечивает достаточного качества для финальной модели.

Основные проблемы:

1. После удаления пересечений с golden set осталось только 251 train-пример.
2. В train полностью отсутствуют 6 из 13 классов.
3. Подбор thresholds делался на validation из 51 примера, где у многих классов support равен нулю.
4. Модель сильно завышает число предсказанных labels.
5. Exact match на golden set равен 0 из 761.
6. Лучший macro F1 на golden set после threshold tuning — только около `0.208`.

Результаты показывают необходимость более качественной подготовки данных и более устойчивого подхода к представлению текста. Последующие embedding-эксперименты используют GPT-5-размеченный train set, отдельный manual-check/golden set и более сильные embedding-модели.

# 22. Сравнение двух этапов

## Сравнение лучших моделей

| подход | лучшая конфигурация | macro_precision | macro_recall | macro_f1 | micro_f1 | weighted_f1 |
|---|---|---:|---:|---:|---:|---:|
| Embedding + classifier | bge-m3 + LinearSVC_balanced__thresholded | 0.820 | 0.833 | 0.822 | 0.829 | 0.829 |
| RuBERT frozen encoder | RuBERT_frozen_head__thresholded_on_val | 0.510 | 0.701 | 0.564 | 0.573 | 0.556 |
| RuBERT head-only | tuned thresholds | 0.174 | 0.462 | 0.208 | 0.285 | 0.272 |

## Главный вывод

Embedding-подход превосходит RuBERT frozen encoder и RuBERT head-only по всем основным метрикам:

```text
macro_f1:    0.822 vs 0.564
micro_f1:    0.829 vs 0.573
weighted_f1: 0.829 vs 0.556
```

Причины:

1. Готовые embedding-модели лучше подходят для sentence-level семантики отзывов.
2. Линейные классификаторы хорошо работают поверх качественных эмбеддингов.
3. RuBERT использовался в режиме frozen encoder, то есть его веса не дообучались под конкретные классы.
4. Размер train set относительно небольшой, поэтому полноценное обучение нейросетевой модели сложнее.
5. Классы пересекаются по смыслу, и линейный classifier поверх сильных эмбеддингов оказался устойчивее.

3. Попробовать полноценный fine-tuning RuBERT или другого русского encoder, а не только frozen encoder.
4. Попробовать ансамбль: например, оставить `bge-m3 + LinearSVC` как основную модель, но для отдельных классов использовать модели, которые дали лучший per-class F1.
5. Перепроверить threshold tuning на более крупном validation/golden set, потому что на маленькой выборке пороги могут быть нестабильны.

# 25. Краткое резюме

В работе были сравнены два подхода к multi-label классификации отзывов: классификация поверх готовых embedding-моделей и классификация на базе замороженного RuBERT encoder. Лучший результат показала связка `bge-m3 + LinearSVC_balanced` с индивидуальными порогами по классам: macro F1 = 0.822. RuBERT frozen encoder дал существенно более низкое качество: лучший macro F1 = 0.564. Поэтому для финального решения был выбран embedding-подход, так как он лучше балансирует precision и recall по всем проблемным классам и устойчивее работает на golden set.
